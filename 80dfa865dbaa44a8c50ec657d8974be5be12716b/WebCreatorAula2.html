>Introduction to Git and Linux

What is Linux?

Just like Windows, iOS, and Mac OS, Linux is an operating system. In fact, one of the most popular platforms on the planet.
Android is powered by the Linux operating system.
An operating system a is software that manages all of the hardware resources associated with your desktop or laptop.

WHY DO I NEED IT?

  • High performance computing
  • Remote access
  • Simple interface
  • Process automation & reproducibility

What is WSL?

The Windows Subsystem for Linux (WSL) lets developers run a GNU/Linux environment directly on Windows, unmodified, without the overhead of a traditional virtual machine or dualboot setup

WHY DO I NEED IT?

  • Command-line tools
  • Utilities
  • Applications like Git for example…

What is Git?

Git was written by Linus Trovlads(Linux Founder)
Git is a version control system (VCS)

Projects in Git are called Repositories:
  • Contains all files
  • Including the respective edit history
  • Each change is called a commit
  • Linked
  • Organized in branches (like a tree)

WHY DO I NEED IT?

We need Git because we’re going to store our WebSite in GitHub Pages services
GitHub Pages is a site hosting service that takes HTML, CSS, and JavaScript files straight from a repository on GitHub and publishes the website on the web.

list

Get started with Git

First of all go to GitHub.com and create and account
Create an empty Repository inside GitHub
Congrats now you have GitHub Repositor

Now lets "link" your PC to GitHub:

PHASE-00

:: How to configure git

git config --global user.name "YOUR USER NAME"
git config --global user.email "YOUR EMAIL"

PHASE-01

:: How to do a first “push” to GitHub

Go to the folder that contain the source code

Create a git repository in your pc

$ git init

PHASE-02

:: Add your files to git
Put your all the files in git

$ git add .
$ git commit -m "Example"

PHASE-03

:: Do a first “push” to GitHub

$ git remote add origin master https://github.com/___.git
$ git push -u origin master

next time you “push” (upload) just type:

$ git pull https://github.com/___.git
$ git push -u origin master

next time you “pull” (download) just type:

$ git pull https://github.com/___.git

NOTE: You must be in the FOLDER_BASE (.git) to pull or push your website correctly

Ok now you have the files for the website in GitHub repository right?
Just go to Pages inside your GitHub repository and get the link to your WebSite
